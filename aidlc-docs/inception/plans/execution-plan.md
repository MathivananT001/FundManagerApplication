Stages to Execute
Stage 1: Application Design — REQUIRED
Rationale: The application is a medium-high complexity full-stack system with:

Hybrid AWS compute strategy (Lambda + ECS Fargate + EC2)
Dual-database architecture (RDS MySQL + DynamoDB)
Role-based access with AWS Cognito multi-method auth
Auction engine, financial ledger, and Bot Agent automation
Flutter Android mobile client with BLoC/Provider state management
A thorough application design is critical to define service boundaries, API contracts, data models, and component relationships before unit decomposition.

Stage 2: Units Generation — REQUIRED
Rationale: The system spans multiple distinct domains (user management, group management, auction engine, financial ledger, notifications, reporting, bot agent). Units generation will decompose the application into implementable modules with clear boundaries, mapped to user stories and implementation phases.

Stage 3: Inception Completed — REQUIRED
Rationale: Final validation checkpoint to confirm all inception artifacts are consistent and ready for the implementation workflow.
